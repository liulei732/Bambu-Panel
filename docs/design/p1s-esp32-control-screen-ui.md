# 拓竹 P1S ESP32 主控制屏 UI 设计

日期：2026-05-24

## 目标

为一块 800x480 横屏电容触摸 ESP32 开发板设计拓竹 P1S 主控制屏 UI。它的定位是机器旁边的完整控制面板，而不是单纯的状态显示屏。

## 已确认范围

- 打印机：Bambu Lab P1S。
- 屏幕：800x480 横屏，电容触摸。
- 屏幕角色：主控制屏。
- 连接方式：ESP32 在局域网内直连打印机。
- AMS：2 台 AMS，共 8 个料槽。
- 首页预览：默认显示打印进度和文件缩略图；摄像头放在二级任务详情页。
- 文件页：第一版显示当前任务和最近任务，保留后续完整文件浏览与发起打印入口。
- 视觉风格：深色拓竹控制台风格，使用青绿色强调色，危险操作使用克制的红色。

## 信息架构

左侧固定导航栏包含 6 个入口：

1. 首页
2. 文件
3. 控制
4. AMS
5. 维护
6. 设置

左侧导航在所有一级页面保持可见，用户可以一键切换主要功能。当前页面使用青绿色强调条和更强的背景色标识。

## 首页

首页回答两个问题：打印机现在怎么样，用户马上能做什么。

首页内容：

- 顶部状态栏：打印机名称、当前状态、局域网连接、AMS 数量、时间。
- 主任务面板：文件名、缩略图或稳定预览、大号进度百分比、进度条、剩余时间。
- 温度摘要：喷嘴和热床的当前温度与目标温度。
- AMS 摘要：8 个料槽、当前使用槽、颜色色块。
- 底部快捷按钮：暂停/继续、速度、风扇、灯光、停止。

快捷按钮需要足够大，适合站在机器旁边快速触摸。停止按钮使用危险色，并且必须二次确认。

## 文件页

第一版以稳定显示任务信息为主，不强做完整文件管理。

文件页内容：

- 当前任务卡片：文件名、状态、可用时显示缩略图、已用时间、剩余时间、可用时显示耗材需求。
- 最近任务列表：文件名和最终状态。
- 后续功能入口：完整 SD 卡或缓存文件浏览、从屏幕发起打印。

这样第一版更稳，也为后续扩展留下明确位置。

## 控制页

菜单名称定为“控制”，而不是“温控”，因为它不仅管理温度，也管理运行参数。

控制页内容：

- 喷嘴目标温度。
- 热床目标温度。
- 风扇控制。
- 速度倍率。
- 流量倍率。
- 耗材预热配置，例如 PLA、PETG、ABS、自定义。

数值控制使用“大号步进器 + 滑条”的组合。常用调整用 `-5` 和 `+5`，点击数值可以直接输入目标值。

## AMS 页

AMS 页内容：

- 2 个 AMS 分组，每组 4 个槽位。
- 槽位颜色、材料类型、槽位编号、已加载/空槽状态。
- 当前使用槽和当前耗材路径。
- 在打印机状态允许时提供装料、退料、换料入口。
- 显示卡料、空槽、材料不匹配、AMS 不可用等错误状态。

8 个槽位需要在 800x480 屏幕内完整可见，不依赖横向滚动。

## 维护页

维护页放低频但重要的机器动作：

- 归零。
- 校准入口。
- 挤出和回抽。
- 清洁喷嘴辅助。
- 首页快捷按钮之外的灯光控制。

会移动硬件或影响打印关键状态的动作，需要和普通开关在视觉上区分。

## 设置页

设置页内容：

- 打印机连接：IP 地址、LAN Access Code、连接测试。
- 局域网状态。
- 屏幕亮度。
- 语言。
- 主题选择。
- 重置本屏幕本地配置。

重置或覆盖配置必须二次确认。

## 全局状态

UI 至少支持这些打印机状态：

- 空闲
- 准备中
- 打印中
- 暂停
- 完成
- 错误
- 离线

错误和离线状态会覆盖首页主要区域，显示明确的恢复面板。恢复面板提供重连、查看详情，以及在合适场景下关闭提示。

## 交互规则

- 危险操作需要二次确认：停止打印、重置屏幕配置、会覆盖已保存连接信息的重新连接。
- 普通操作不需要二次确认：暂停/继续、灯光开关、速度调整、风扇调整、预热选择。
- 点击首页温度摘要进入“控制”页。
- 点击首页任务预览进入“任务详情”页。
- 任务详情页可以尝试显示摄像头。如果摄像头流不可用，或对 ESP32 压力过大，则降级为缩略图、进度和状态日志。

## 视觉方向

采用深色工业控制台风格：

- 背景：接近黑色的深蓝。
- 主面板：深蓝灰。
- 强调色：青绿色，用于当前状态、选中态、进度。
- 警告色：琥珀色。
- 危险色：红色，只用于停止、重置等高风险动作。
- 字体：紧凑、清晰的无衬线字体；不使用随视口缩放的字体。
- 圆角：克制的小圆角，约 6-8 px。

整体需要耐看，适合长期放在打印机旁边。

## 布局约束

- 目标分辨率固定为 800x480。
- 导航、快捷按钮、AMS 槽位、数值控制都需要稳定尺寸，避免动态内容导致布局跳动。
- 不使用卡片套卡片。
- 避免纯装饰性渐变或影响可读性的光效。
- 中文标签必须能完整显示，不截断。
- 高频和高风险操作必须保留足够大的触摸区域。

## 第一版不做

- 完整 SD 卡或缓存文件浏览。
- 从屏幕发起任意文件打印。
- 首页强制加载实时摄像头。
- 依赖云端的流程。
- 多打印机仪表盘。

## 实现备注

- UI 需要让数据更新和页面导航相互独立。
- 当缩略图、最近文件、摄像头等可选数据缺失时，首页仍然可用。
- 连接失败和打印机命令失败必须显示明确状态，不能静默失败。
- 设计应兼容 LVGL 风格控件和 ESP32 内存约束。
